Layout

Hexo 有三种默认布局: post, page 和 draft. 三种不同的文件分别保存在不同路径. 新创建的post文章保存在source/_post 文件夹里.

Layout Path
post source/_posts
page source
draft source/_drafts

不要处理我的文章(post)
如果你不想文章被处理，你能在yaml头部设置 layout: false

文件名

Hexo 默认使用文章题目作为文件名. 你可以编辑_config.yml中的new_post_name设置来改变默认的文件名. 例如，`:year-:month-:day-:title.md 可以设置为文件名的格式以创建日期为前缀. 你能使用如下占位符:

Placeholder Description
:title 文章名字(小写， ‘-‘代替空格)
:year 创建年
:month 创建月(两位数)
:i_month 创建月(没有开头的零)
:day 创建日(两位数)
:i_day 创建日(没有开头的零)

草稿

先前, 我们提到过一个特别的layout: draft. 草稿被保存在 source/_drafts中. 你能使用publish命令把草稿移动到source/_post中, publish 命令跟new命令相似.

草稿默认不会展示, 你能在运行Hexo时加上--draft选项或者在_config.yml中开启render_drafts设置来渲染草稿.
